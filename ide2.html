Atom

Splitting the Atom

Dans une approche incorporant un code open-source, gratuité, et librement customisable en HTML, CSS et Javascript, Atom était une promesse d'accessibilité. Sorti de sa version Beta en 2015, il vit aussi apparaître un challenger du nom de Visual Code.
Le début de la chute.

la fuite des cerveaux

Sur ce début prometteur, mettant en avant une communauté très présente, les choses se sont d'autant plus gâtées lorsque Microsoft, à racheté Github, pour une somme que la décence m'interdit de reproduire.
Rien n'est La récente annonce d'un EDI développé directement par Github peut évoquer à certains, un stade de mort annoncée pour Atom (source).

En résumé

Avantages:

  • Disponible sur toutes les plateformes.
  • Communauté active sur les forums.
  • Package manager incorporé.
  • Auto-completion bien pensée.
  • Possibilité d'utiliser sans limite de temps.

Inconvénients:

  • Développement presque à l'arrêt depuis le rachat de Github.
  • Fin de l'Eldorado pour les développeurs qui ont depuis migré vers Visual Code
  • Difficultés lors de l'intégration de git...
  • Agonisant en comparaision à Visual Code
